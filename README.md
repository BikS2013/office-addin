# Word Add-in API Sidebar

A generic Microsoft Word add-in implemented as a sidebar (task pane) that dynamically loads API configurations from a remote URL, renders API groups as a hierarchical tree, and allows users to execute API calls with optional prompts combined with selected text or full document text.

## Features

- **Dynamic API Configuration** — Load API definitions from any HTTPS URL or local JSON file
- **Hierarchical API Tree** — Organize APIs into nested groups displayed as a collapsible sidebar tree
- **Text Extraction** — Extract selected text or full document content from the active Word document
- **Prompt Templates** — Pre-configured prompt templates with `{{text}}`, `{{prompt}}`, and `{{documentName}}` placeholders
- **Flexible Input Modes** — Per-API control over whether to send selected text, full document, or let the user choose
- **Response Handling** — Display API responses with dot-notation field extraction, copy to clipboard, and insert into document
- **Execution History** — Persistent history of all API interactions stored in IndexedDB with filtering and replay
- **Configuration Caching** — Cached configs with 1-hour TTL to minimize network requests

## Prerequisites

- **Node.js** 18+ and npm
- **Microsoft Word** (Windows, macOS, or Office for the web)
- HTTPS-enabled API endpoints with CORS support

## Quick Start

```bash
# Install dependencies
npm install

# Launch dev server and sideload in Word
npm start

# Run tests (83 tests)
npm test

# Production build
npm run build

# TypeScript type checking
npm run lint
```

The dev server starts on `https://localhost:3000`. HTTPS certificates are auto-generated by `office-addin-dev-certs`.

## Architecture

| Layer | Technology |
|-------|-----------|
| Runtime | Office.js Word API (WordApi 1.3+) |
| Framework | React 18 + TypeScript (strict mode) |
| UI Library | Fluent UI React v9 |
| State Management | React Context + useReducer |
| Persistent Storage | IndexedDB (history) + localStorage (config cache) |
| Build | Webpack 5 with HTTPS dev server |
| Tests | Vitest |

## Configuration

The add-in loads its API configuration at runtime from a user-provided HTTPS URL. No configuration is hardcoded.

A sample configuration is available at [`docs/reference/sample-config.json`](docs/reference/sample-config.json).

For the full configuration schema, caching behavior, localStorage keys, IndexedDB settings, and CORS requirements, see the [Configuration Guide](docs/design/configuration-guide.md).

**Policy:** Missing required configuration fields raise exceptions. No fallback or default values are ever substituted.
